# 六點把手區塊型態選單設計

## 背景

目前 Crepe 的六點把手主要用於拖曳區塊。既有 Slash 選單只適合在空白區塊輸入 `/` 後插入或建立內容；若直接用它處理已有文字的區塊，Crepe 會將整段文字當成命令搜尋條件並關閉選單，因此不能可靠地用來轉換目前區塊。

本功能讓使用者點擊六點把手後，直接變更目前區塊型態。例如將已有內容的 H1 轉成 H2，且不新增空白區塊。

## 目標

- 點擊六點把手時，開啟目前區塊的型態轉換選單。
- 轉換後保留目前區塊的文字內容。
- 保留六點把手原有的拖曳功能。
- 不新增第三方套件或外部資源。
- 建置成品仍可在 Chrome 147.0.7727.102 中以 `file://` 完全離線執行。

## 選單範圍

選單只提供可作用於目前文字區塊的型態：

- 一般文字
- Heading 1
- Heading 2
- Heading 3
- Heading 4
- Heading 5
- Heading 6
- 引用
- 程式碼區塊

程式碼區塊預設不指定語言。本功能不加入清單、表格、圖片、分隔線或其他插入型命令，也不提供程式碼語言選擇器。

## 互動行為

1. 游標移到支援轉換的文字區塊時，沿用現有六點把手。
2. 使用者按下並放開六點把手，且指標移動距離未超過既有點擊門檻時，在把手旁開啟區塊型態選單。
3. 選單標示目前區塊型態。
4. 選擇另一型態後，原地轉換目前區塊並關閉選單。
5. 點擊選單外、按下 `Escape` 或開始拖曳時關閉選單。
6. 指標移動超過點擊門檻時視為拖曳，不開啟選單。

選單外觀沿用現有編輯器的顏色、圓角、陰影及選單項目尺寸，但不依賴 Crepe Slash 選單的私有狀態或 DOM 結構。

## 內容與格式規則

- 一般文字與 Heading 1–6 互轉時，保留文字與可由目標節點承載的行內格式。
- 轉成引用時，以引用節點包住目前文字區塊；由引用轉成其他型態時，先解除目前引用層，再套用目標型態。
- 轉成程式碼區塊時保留純文字。粗體、斜體、連結等行內格式無法存在於 Markdown fenced code block，因此會移除其格式語意。
- 由程式碼區塊轉回一般文字、標題或引用時保留純文字，不復原先前移除的行內格式。
- 選擇目前已套用的型態不修改文件，只關閉選單。
- 不清空目前區塊，也不建立額外空白區塊。

## 實作邊界

### 區塊選取與點擊判定

由 `EditorManager` 接收六點把手的指標事件，區分點擊與拖曳，並在點擊當下記錄目前區塊位置。不得覆蓋 Crepe 原有的拖曳處理器。

### 型態選單

以專案內的 DOM、CSS 與事件處理建立小型選單。選單只接收目前型態、定位資訊及選取回呼，不直接管理文件內容。選單銷毀時必須移除自身事件監聽器與 DOM。

### 區塊轉換

透過現有 Milkdown／ProseMirror schema 與 command API 轉換目前區塊：

- 一般文字、Heading 1–6、程式碼區塊使用 block type 轉換。
- 引用使用 wrap／lift 操作，避免建立巢狀引用。
- 執行轉換前重新驗證已記錄位置仍對應可轉換區塊；若文件已改變或位置失效，關閉選單且不修改文件。

不呼叫 Slash 命令使用的 `clearTextInCurrentBlockCommand`，因為該命令會刪除目前文字。

## 錯誤與邊界處理

- 表格、圖片、水平分隔線等不支援的節點不開啟此選單。
- 選單開啟後若目前區塊被其他操作刪除或改變，取消轉換。
- 編輯器銷毀或重新建立時，一併關閉選單並移除監聽器。
- 任何不能安全套用的轉換都維持原文件內容，不退回成插入空白區塊。

## 驗證標準

### 自動化測試

- H1 轉成 H2，原文字保持不變且不留下 H1。
- H2 轉成一般文字，原文字保持不變。
- 一般文字轉成引用，再轉回一般文字，不產生巢狀或多餘區塊。
- 一般文字轉成程式碼區塊，純文字保持不變。
- 含行內格式的文字轉成程式碼區塊時，文字保留且不殘留不支援的 marks。
- 選擇目前型態不產生文件變更。
- 拖曳手勢不開啟型態選單。
- `Escape` 與點擊選單外可關閉選單。

### Chrome UI 驗收

- 使用 repo 內既有 Chrome 147.0.7727.102 執行 `release/markdown-editor.html`。
- 以 `file://` 開啟單檔成品。
- 驗證六點把手可將 H1 轉成 H2 並保留內容。
- 驗證可轉成程式碼區塊。
- 驗證完整 UI 測試沒有回歸。
- 測試期間攔截 HTTP／HTTPS，確認沒有外部網路請求。

## 非目標

- 不修改鍵盤輸入 `/` 的既有 Slash 選單行為。
- 不新增十種主題切換。
- 不新增套件、CDN、線上服務或執行時網路需求。
- 不要求 Windows 11 實機驗收或 Chrome 150 測試。
- 不進行與本功能無關的安全性檢查或重構。
